Macro View Variables

Use the Macro Variables dialog to tag and watch the values of user, or internally, defined variables change while a macro is playing. This can be very useful when you are debugging a macro. Tagging a specific variable turns it into a "Watch Variable."

 

The following areas are provided by the Macro Variables dialog:

  • User-Defined or Internal Variable: These text fields, located at the top of the Macro Variables dialog, allows you to specify and view the changing value of the user, or internally, defined variables.

 

To specify a variable to watch:

  1. Click on one of the User-Defined or Internal Variable text fields.

  2. Type in the variable name.

 

This tags the variable. Then you can move this dialog to the side and run your macro. An alert dialog is displayed whenever the tagged variable is accessed by the macro. The alert lets you know that the value is changing. Click OK, and look at the Value field in the Macro Variables dialog.

 

The Macro Variables dialog also automatically displays the values of any loops and the parameter values of any macro function calls to the stack as the macro is playing. The following areas in the Macro Variables provide the loop and parameter viewing:

  • Loops: When the macro viewer evaluates a loop macro command it automatically displays the current iteration value in the Value text fields. The total iterations value of that loop are displayed in the End Value text fields.

  • Call Stack: When the macro viewer evaluates a nested macro command, the parameter values that the nest macro is called with are displayed in the Call Stack parameter text fields.

 

Note: All the text fields in this dialog are editable, including the value text fields. This means you can change the value of a watch variable or parameter value as the macro is running to correct a problem or test a situation.
