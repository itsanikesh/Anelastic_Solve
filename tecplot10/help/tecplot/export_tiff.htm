Export TIFF

Export a Tagged Image File Format (TIFF) file.

 

You may select Color or Gray Scale. For gray scale, you may select an image depth (see below for details). The size of the image can be controlled with the Width option.

 

The following options are available:

  • Color: Select this check box to save the image as a color TIFF file.

  • Region: Select Current Frame, All Frames, or Work Area from the drop-down. Current frame is the default and captures only the current frame. All Frames captures the smallest rectangular area containing all frames. Work Area captures the workspace.

  • Use Width of Image on Screen: Select this option to generate an image file the same size as the current plot on the screen. This option is forced on if you use on-screen image rendering in the Display Performance dialog.

  • Enter Width (pixels): Enter a value in the text field for your exported image’s width. The image region is rendered to the image file to exactly fit a size of Width by Height. (Default is 300.)

  • Antialiasing: Select this option to remove jaggies from the image.

  • Supersample Factor: Control the amount of antialiasing used in the image.

  • Convert to 256 Colors: Selecting this check box will convert the true color screen image to eight-bit color.

  • Depth: Available only when the Color check box is not selected. Choose from On/Off, 1-Bit/Pixel, 4-Bit/Pixel, or 8-Bit/Pixel.

  • On/Off uses one bit per pixel, with the pixel off for black, and on for any other color.

  • One-Bit/Pixel uses one bit per pixel, with the pixel off if the gray scale equivalent of the color is less than 50 percent white, on if more than 50 percent.

  • Four-Bit/Pixel uses four bits per pixel to allow moderate gray scale.

  • Eight-Bit/Pixel uses eight bits per pixel to allow full gray scale.

Note: All TIFF images may either be eight-bit or true color. All TIFF images are compressed.

 

Performance Tips

  • If exporting is taking an unusually long time, or you get an error message saying that the image cannot be exported, the most likely cause is that the image width you are trying to export is too large. Selecting a smaller image width will greatly speed up the export process.

  • Note that for an image export size of Length x Width, the file size for an uncompressed true color image is approximately Length x Width x 3. Memory requirements to export such an image can be up to twice this size.

  • For 256 color images, the maximum file size is approximately Length x Width, but is usually less since all 256 color image files are compressed. However, the memory requirements for exporting are the same as they are for a true color uncompressed image.
