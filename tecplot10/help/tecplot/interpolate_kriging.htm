Interpolate Kriging

Use the Kriging dialog to perform kriging, a more complex form of interpolation than inverse-distance. It works like inverse-distance interpolation and is used for the same purposes. Kriging generally produces superior results to the inverse-distance algorithm but requires more computer memory and time. Tecplot uses the current frame's axis assignments to determine the variables to use for coordinates in kriging, but it ignores any axis scaling that may be in effect.

 

The following options are available:

  • Source Zones: Select the zone or set of zones to krig from (that is, the zone[s] with the source data).

  • Variables: Select the variables to krig. By default, all variables (other than those assigned to the X-, Y-, and Z-axes) are kriged. Kriging fewer variables can speed up the process.

  • Destination Zone: Select the destination zone into which to krig. Except for the variables assigned to the axes, data in the destination zone is overwritten.

  • Range: Select the distance beyond which source points become insignificant for the kriging. The value is stated as the fraction of the length of the diagonal of the box which contains the data points. A range of zero means that any point not coincident with the destination point is statistically insignificant; a range of one means that every point in the data set is statistically significant for each point. In general, values between 0.2 and 0.5 should be used.

  • Zero Value: Enter the semi-variance at each source data point on a normalized scale from zero to one. Semi-variance is the certainty of the value at a data point. A value of zero means that the values at the source points are exact. Greater values mean the values at the source points have some uncertainty or noise. Zero is usually a good number for Zero Value, and it causes the interpolated data to fit closely to all the source data points. Increasing the Zero Value results in smoother interpolated data that fits increasingly more to the average of the source data points.

  • Drift: Select the overall trend for the data from the drop-down. The available choices are No Drift, Linear, and Quadratic. If the Drift is set to Linear or Quadratic, the selected points must be non-collinear (non-coplanar in the 3D Cartesian plot type).

  • Point Selection: Select from the drop-down the method used for determining which source points to consider for each destination point. This option is very important for kriging, since kriging involves the computationally expensive inversion and multiplication of matrices. The computational time and memory requirements increase rapidly as the number of selected source data points increases. In general, you should not use All unless you have very few source points, or extra time and computer memory. There are three available methods, as follows:

  • Nearest N: For each point in the destination zone, consider only the closest n points to the destination point. These n points can come from any of the source zones. You specify n after selecting this option. This option may speed up processing if n is significantly smaller than the entire number of source points.

  • Octant: Like Nearest N above, except the n points are selected by coordinate-system octants. The n points are selected so they are distributed as evenly as possible throughout the eight octants. This reduces the chances of using source points which are all on one side of the destination point.

  • All: Consider all points in the source zone(s) for each point in the destination zone. Keep in mind that points far away have a very small weighting factor.

  • Number of Points: Enter the number of points n to consider for each destination point. This field is enterable and used only if the Point Selection is set to Nearest N or Octant. This field is ignored if Point Selection is set to All.
